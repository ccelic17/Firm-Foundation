What Is This App

Firm Foundation is a faith-based personal development PWA built for Christian men who want a single daily command center — Spirit, Mind, and Body — with accountability at the core. The tagline "Don't Drift" captures the mission: a daily system to keep you anchored to your non-negotiables when life tries to pull you off course.

The existing prototype is a polished single-file HTML proof-of-concept with all core screens designed and data flowing through localStorage. This PRD plans the transition from prototype to a real, shipped product — starting with the fastest possible win and layering complexity from there.

Data Persistence

ToolKey FactsPhase
Dexie.js Minimal IndexedDB wrapper. Same mental model as localStorage but 50MB+. Used by WhatsApp Web, GitHub Desktop. Zero config. Phase 1
Supabase PostgreSQL backend + auth + real-time subscriptions + edge functions. Open-source, self-hostable. Best for complex queries (90-day goals, prayer history). Phase 2
RxDB Local-first reactive database. Syncs local IndexedDB → Supabase/Firebase. Offline-first with cloud backup. Phase 2+

Streak Tracking

ToolKey FactsCost
@marekzelinka/streak-counter TypeScript npm package. localStorage-based. Duolingo-inspired (current / longest streak). ~2–3 KB. Drop-in replacement for existing streak logic. Free / OSS

Goal Decomposition

No production-ready library exists for faith-and-fitness goal decomposition. The best approach is Claude Sonnet with structured JSON output — which is exactly what the prototype stubs. The system prompt defines the shape: 3 phases (30/30/30 days), milestones, and 4 daily actions for the current week. No external library needed.

Decisions Needed Before Phase 1

1. API key architecture — proxy or client-side?
Phase 1 calls to Anthropic can go client-side (matching the prototype pattern) or through a Netlify Function proxy to keep the key server-side. The proxy is the right call for production but adds one build step. Recommendation: proxy via Netlify Function from day one.
2. Default Bible translation?
bible-api.com supports ESV, KJV, NIV, NLT, CSB, and others. The app needs a default. Recommendation: ESV (widely used in evangelical/Reformed circles, clean language, free for non-commercial use).
3. Streak grace day mechanic?
Currently the streak resets if you miss a single day (existing prototype behavior). Options: (a) hard reset — miss = zero, (b) one grace day per 7-day period (Duolingo model), (c) no grace, but show "longest streak" as the legacy metric. Recommendation: no grace day — the whole brand is "Don't Drift," softening that undermines it.
4. AI cost control on the Goal Engine?
Generating a 90-day plan with Claude Sonnet uses ~1,500 tokens (~$0.005/call). At low user counts this is negligible, but should regeneration be rate-limited? Options: (a) unlimited, (b) once per 7 days, (c) unlimited in Phase 1, rate-limit at scale. Recommendation: unlimited in Phase 1 to reduce friction during early validation.

Risks & Mitigations

Risk
WHOOP rate limits on sync frequency
Mitigation
Cache recovery score locally; cap syncs at once per hour max. Existing prototype already models this behavior.
Risk
Garmin API approval timeline (can take weeks or months)
Mitigation
Phase 2 ships with WHOOP-only. Garmin is additive — apply for enrollment now so approval arrives before Phase 2 wraps.
Risk
Apple HealthKit has no web API
Mitigation
Clearly marked "coming soon" in UI. Defer to Phase 3 native shell. Don't promise in Phase 1 or 2 marketing.
Risk
Anthropic API cost at scale
Mitigation
Use Haiku for short prompts (prayer, verse search). Use prompt caching for repeated system prompts. Monitor usage in dashboard from day one.
Risk
bible-api.com reliability / downtime
Mitigation
Cache today's verse in Dexie.js. On fetch failure, serve yesterday's cached verse. Fallback pool of 30 hardcoded verses for complete offline use.
Risk
PWA push on iOS requires home screen install
Mitigation
In-app "Add to Home Screen" prompt on first use. iOS 16.4+ supports web push for installed PWAs. Phase 2 feature — not Phase 1.
